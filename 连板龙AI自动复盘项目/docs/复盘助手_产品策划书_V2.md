# 连板龙 — 产品策划书 V2

> **产品形态**：交易软件自动复盘插件  
> **交付方式**：网页下载 + 本地安装  
> **收费模式**：年订阅制  
> **版本**：V2.0  
> **日期**：2026-06-01

---

## 一、产品定位

### 一句话定义

> 每天收盘后自动更新交易软件自定义板块，用户打开软件即可看到涨停梯队、龙虎榜、热点概念全景，无需人工筛选。

### 用户痛点

| 痛点 | 本产品方案 |
|:----|:----------|
| 收盘后需手动翻涨停列表，分类连板 | 自动获取全部涨停数据，按连板天数分类 |
| 不知道今天哪个概念最强 | 自动计算概念涨停数排名，标识主线/次主线/潜在 |
| 龙虎榜需切换网站查看 | 自动拉取写入同一板块 |
| 不知道昨天涨停的今天表现如何 | 自动统计晋级率/溢价率/断板率 |
| 复盘工作重复枯燥 | 全自动，打开软件直接看 |

## 四、算法与标准

### 4.1 连板天数计算

```
输入：当日涨停股票列表 + 前5个交易日涨停数据
算法：
  for each 涨停股票:
    天数 = 1
    for each 历史交易日:
      if 该股在历史日也涨停:
        天数 += 1
      else:
        break  # 连板断了
输出：每只股票的连板天数
```

**关键规则**：
- 每只股票独立计算，互不影响
- 只遍历有交易数据的日期，自动跳过非交易日
- ST股涨停标准为5%，正常股为10%

### 4.2 热点概念排名

```
输入：zzshare uplimit_hot 数据
      每只涨停股票的所属概念列表
算法：
  for each 涨停股票:
    for each 所属概念:
      该概念涨停数 += 1
  
  按涨停数降序排列
  涨停数相同的，按zzshare热度分排序
  
  主线 = 第1名
  次主线 = 第2名（排除已入主线的股票）
  潜在 = 第3名（排除已入主线+次主线的股票）
```

**关键规则**：
- 只有有涨停的概念才参与排名
- 三个概念板块互斥，股票不重叠
- 概念名通过映射表转为市场公认名

### 4.3 概念板块文件生成

```
1. 取热点第1概念的所有涨停股 → 写入 RDGN.blk
2. 取热点第2概念 → 排除已在RDGN中的股票 → 写入 HYRD.blk  
3. 取热点第3概念 → 排除已在RDGN/HYRD中的股票 → 写入 RDGN3.blk
4. 动态更新 blocknew.cfg 和 gridtab.dat 中的概念名
```

### 4.4 安装逻辑

```
1. 检测用户交易软件安装路径
2. 写入 blocknew.cfg（概念名称映射）
3. 写入 gridtab.dat（顶部标签栏）
4. 创建空的 .blk 文件
5. 部署引擎程序
6. 注册定时任务（每天16:00）
7. 首次运行一次引擎获取数据
```

## 五、销售渠道：淘宝

### 5.1 淘宝销售流程

```
用户在淘宝搜索/推荐看到商品
  ↓
在淘宝下单付款（¥99/年）
  ↓
淘宝自动发货（无需人工干预）：
  ├── 第1条信息：产品网页链接 + 激活码
  ├── 第2条信息：使用教程链接
  └── 激活码格式：FP-XXXX-XXXX-XXXX（16位字母数字）
  ↓
用户打开产品网页 → 选择对应交易软件版本 → 下载安装包
  ↓
解压 → 运行安装 → 输入激活码 → 重启交易软件
  ↓
每天16:00自动复盘更新
```

### 5.2 关键设计原则

| 原则 | 说明 |
|:----|:------|
| **零注册** | 用户无需注册账号、无需绑定邮箱手机 |
| **零服务端** | 产品网页是纯静态页面，激活码本地验证 |
| **零人工干预** | 淘宝自动发货，全自动化流程 |
| **完全匿名** | 用户身份仅淘宝订单号知晓，产品内无任何用户信息 |

### 5.3 激活码设计

激活码采用**离线校验方式**，不需要服务器参与：

```
激活码格式：FP-YYYYMMDD-XXXX-XXXXXXXX
  ├── YYYYMMDD = 授权截止日期（如 20270601 = 一年后）
  ├── XXXX = 产品标识（软件版本）
  └── XXXXXXXX = 校验码（基于密钥+日期计算的签名）

校验方式：程序本地解码，无需联网
生成方式：卖家使用离线工具生成，不依赖服务端
```

**激活码示例**：
```
FP-20270601-A001-8F2D9C3B  ← 有效期至2027年6月1日，通达信版
```

**安全性**：虽然客户端可以看到截止日期，但无法伪造有效的校验码（需要密钥）。如果有人公开分享激活码，卖家可以在下一批次更换密钥。

### 5.4 续费机制

续费通过**再次在淘宝购买**实现：

```
用户收到到期提醒（程序检测本地激活码到期）
  ↓
打开淘宝 → 搜索商品 → 购买"续费链接"
  ↓
淘宝自动发货 → 新的激活码（新的截止日期）
  ↓
用户打开程序 → 输入新激活码 → 覆盖旧码
  ↓
恢复更新
```

| 续费场景 | 操作 | 用户体验 |
|:---------|:-----|:---------|
| 到期前提醒 | 程序弹窗提示"距到期还有X天" | 无操作 |
| 已到期 | 数据更新停止，程序提示续费 | 淘宝买→输新码→恢复 |
| 中途续费 | 可提前输入新码，旧码自动覆盖 | 不影响使用 |

## 八、数据更新流程（不变）

### 8.1 每日更新

```
16:00 定时任务触发
  ↓
获取涨停数据（同花顺）
获取龙虎榜数据（ClawHub）
获取封板/炸板数据（东方财富）

完成
```

```
用户双击桌面"连板龙"快捷方式
  ↓
launcher.exe 运行
  ↓
检查是否今天已更新？

  └── 否 → 运行引擎 → 更新数据 → 启动交易软件
  ↓
用户打开软件即可看到最新复盘数据
```

## 九、风险与合规

### 8.1 合规边界

本产品定位为**数据聚合与展示工具**，不是证券投资咨询：

| 可以做 | 不能做 |
|:-------|:-------|
| 展示公开涨停数据 | ❌ 推荐个股 |
| 自动分类连板天数 | ❌ 预测涨跌 |
| 统计概念热度排名 | ❌ 承诺收益 |
| 龙虎榜数据展示 | ❌ 代客理财 |

### 8.2 免责声明

产品安装包和网页中均需包含免责声明：

> 本软件为数据整理工具，仅提供公开市场数据的自动分类与展示功能。不提供任何投资建议、不推荐任何股票、不预测市场走势。用户基于本软件做出的任何投资决策，均由用户自行承担风险。股市有风险，投资需谨慎。

### 8.3 API风险

| 风险 | 概率 | 应对 |
|:-----|:----:|:-----|
| 同花顺API改版 | 低 | 多数据源备选 |
| zzshare服务中断 | 低 | 备用东方财富概念接口 |
| ClawHub不可用 | 中 | 龙虎榜板块暂时为空，不影响其他功能 |
| 通达信版本升级 | 低 | 关注更新，及时适配 |
