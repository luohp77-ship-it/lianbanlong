一、产品概述

1.1 一句话定义

用户下载一个 exe，双击后在浏览器里完成安装配置，每天自动更新通达信板块和可视化复盘看板，收盘后微信推送摘要到手机。

1.2 解决的用户痛点

痛点现状本产品方案
涨停跌停还要手动翻收盘后逐个翻看自动抓取全量数据，分类写入通达信板块
连板数要自己算记不住每个股连板数自动计算首板~5板以上
龙虎榜要切网站看打开东财/同花顺翻自动拉取写入板块+网页可视化
不知道今天热点在哪凭感觉猜板块热度 TOP10 + 概念关系图
不知道昨天涨停今天表现手动翻回去看自动统计晋级率/溢价率/断板率
收盘后不在电脑前收不到复盘信息微信推送复盘摘要到手机
每天复盘工作重复枯燥每天做同一套全自动，打开浏览器看可视化报告

1.3 目标用户画像

维度描述
身份散户股民，有每日复盘习惯

2.2 为什么是"一个exe"

浏览器有安全沙箱，无法直接读写文件系统、调用 Windows API。所以需要一个小型本地服务器来处理系统操作。

❌

纯网页（不行）

  • 展示图表 ✅
  • 用户勾选配置 ✅
  • 写通达信板块文件 ❌
  • 注册定时任务 ❌
  • 替换快捷方式 ❌
✅

一个exe（行）

  • 内嵌 HTTP 服务器 → 网页交互
  • 展示图表 ✅（网页上）
  • 用户勾选配置 ✅（网页上）
  • 写通达信板块 ✅（引擎做）
  • 注册定时任务 ✅（引擎做）
  • 替换快捷方式 ✅（引擎做）
😞

APP（也行，但不够好）

  • 需要双 exe（引擎+配置面板）
  • 可视化能力弱
  • 界面开发成本高
  • 更新配置不方便
  • 不支持图表展示

2.3 技术原理

启动：用户双击 exe → 程序启动内嵌的 HTTP 服务器（端口 8765）→ 自动打开浏览器访问 http://localhost:8765
交互：网页上的每个操作按钮（安装、更新、设置）都通过 fetch('/api/xxx') 调用本地服务器
执行：本地服务器收到请求 → 执行对应的系统操作（写文件、注册任务等）→ 返回结果到网页
数据：引擎每日执行完成后，将数据写入 data/latest.json → 网页读取并渲染图表
后台：exe 缩到系统托盘，保持 HTTP 服务器运行，用户随时打开浏览器查看
与纯网页方案的区别：虽然用户看到的是浏览器页面，但这个页面是由本地的 exe 提供的（localhost:8765）， 不是公网服务器。所以它可以调用本地操作系统的能力。用户不需要联网，也不需要部署服务器。

六、启动机制

6.1 双触发机制

⏰

触发A：定时任务

  • Windows Task Scheduler
  • 每天 15:30 自动运行引擎
  • 电脑开着时到点就更新
  • 无需用户操作
🚀

触发B：通达信启动触发

  • 替换快捷方式 → launcher.exe
  • 检测 last_update.json
  • 需要更新 → 先跑引擎 → 再启动TDX
  • 已更新 → 直接启动TDX（秒开）
🖱️

触发C：手动触发

  • 网页上点击「立即更新」
  • 系统托盘右键「立即更新」
  • 双击桌面快捷方式打开看板
  • 随时可手动运行

6.2 缓存标记

引擎通过检查 data/last_update.json 判断是否需要更新：

{
  "lastDate": "20260529",
  "lastTime": "2026-05-29 15:35:00",
  "status": "success"
}

判断逻辑： 今天 > lastDate → 需要更新 │ 今天 = lastDate → 已最新
