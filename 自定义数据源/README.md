# 通达信自定义数据源 — 涨停原因 + 所属概念

## 数据源

### 涨停原因（数据号 1）
- **来源**：同花顺 zzshare API（`review_uplimit_reason`）
- **内容**：今日涨停的真实原因，如 "AI PC+印制电路板"
- **覆盖**：100% 涨停股
- **依赖**：`pip install zzshare`

### 所属概念（数据号 2）
- **来源**：东方财富板块成分股映射缓存
- **内容**：个股所属概念板块，如 "CPO概念 | 光通信模块 | PCB"
- **覆盖**：约 30~50%（依赖缓存，每天早上自动刷新）
- **依赖**：东方财富 push2 行情API（只拉取一次，缓存在本地）

## 文件说明

| 文件 | 说明 |
|------|------|
| `tdx_zt_monitor.py` | 主脚本，运行后每分钟自动刷新数据 |
| `启动涨停原因监控.bat` | 一键启动批处理 |
| `tdx_concept_cache.json` | 概念板块缓存文件（每日自动刷新） |
| `tdx_zt_monitor.log` | 运行日志 |

## 前置准备（仅首次）

### 1. 安装依赖
```bash
pip install zzshare
```

### 2. 通达信内设置
打开通达信：
```
功能 → 公式系统 → 自定义数据管理器
```
新建两条：
- **数据号 1**：名称 `涨停原因`
- **数据号 2**：名称 `所属概念`

### 3. 行情列表加列
在行情列表（如 401 排序页）表头右键：
```
选择自定义数据 → 勾选「涨停原因」和「所属概念」
```

## 运行

双击 `启动涨停原因监控.bat` 即可。

脚本会自动：
1. 将两列的字体颜色设为荧光绿（修改 `datacfg.dat`）
2. 每分钟从同花顺获取涨停数据
3. 写入 `T0002/signals/extern_user.txt`
4. 通达信表头自动刷新

## 数据文件位置

```
T0002/signals/
├── extern_user.txt    ← 自定义数据内容（脚本自动写入）
├── datacfg.dat        ← 自定义数据配置（含颜色设置）
```

## 颜色

两列字体颜色自动设为荧光绿 `RGB(0, 255, 0)`，通过修改 `datacfg.dat` 中偏移 0x30 的 COLORREF 字段实现。
如需修改，在通达信的自定义数据管理器中右键数据项可调整。
