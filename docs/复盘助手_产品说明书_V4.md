# 连板龙 — 产品说明书 V4

> **产品形态**：通达信AI智能复盘插件
> **交付方式**：本地安装 + 单次定时更新 + 单文件 .exe 加密交付
> **收费模型**：按年订阅制（30日试用低价版）
> **授权说明**：每个激活码最多绑定 2 台设备
> **到期行为**：到期自动降级为试用版，每日更新暂停
> **版本**：V4.0（规划中）
> **状态**：待实现
> **日期**：2026-06-04

## 一、产品定位

### 一句话定义

> 一个开箱即用的通达信复盘插件，每天收盘后自动更新自定义板块，用户打开通达信即可看到涨停梯队、龙虎榜、热点概念全景。

### 解决的用户痛点

| 痛点 | 本产品方案 |
|:-----|:----------|
| 收盘后需手动翻涨停列表、分类连板 | 自动获取全部涨停数据，按连板天数分类写入板块 |
| 不知道今天哪个概念最强 | 综合评分排名（涨停数+强度+资金），动态命名热点概念 |
| 龙虎榜需切换网站查看 | 自动拉取写入同一板块 |
| 跌停/炸板/曾涨停容易忽略 | 自动监控并分类写入对应板块 |
| ST股涨停被主流数据源遗漏 | 双数据源互补：同花顺(非ST) + 东方财富(ST) |
| 看不到涨停原因和所属概念 | 自定义数据列：extern_user.txt 写入涨停原因+所属概念+龙虎榜 |
| 没有系统化复盘报告 | 每天自动生成Word报告，含封面+情绪+涨停+龙虎榜+梯队诊断 |
| 复盘工作重复枯燥 | 全自动，打开通达信直接看 |
| 错过收盘推送 | 微信绑定后每天收盘自动推送关键数据 |

### 目标用户

- A 股散户，有每日复盘习惯
- 使用通达信（含券商定制版）
- 关注涨停板、连板效应、龙虎榜、概念热点
- 使用龙头战法复盘

### 平台策略

**仅支持通达信**。不做东方财富、大智慧等其他交易软件适配。

### 商业渠道

| 渠道 | 用途 |
|:-----|:------|
| **用户中心**（自建网页） | 下载、付款、激活、微信绑定、使用说明、续费 |
| **淘宝/抖音** | 引流（让用户知道有这个产品） |

交易闭环在用户中心内完成，不依赖第三方平台。

### 2.2 标签栏动态命名规则

```
gridtab.dat 名称字段：14字节(GBK)

连板板块：保持固定名（总涨停/首板/2连板/...）
概念板块：{数字}{概念名}，如"1消费电子""2电力改革""3算力通信"
  - 数字1=主线、2=次线、3=潜在
  - 名称不带括号、不带涨停数
  - 长度控制：自动截断确保 ≤14 字节
```

### 2.3 自定义数据列（extern_user.txt）

| 数据号 | 名称 | 内容 | 数据源 |
|:-------|:-----|:-----|:-------|
| 1 | 涨停原因 | 每只涨停股的涨停原因 | 同花顺 reason_type |
| 2 | 所属概念 | 每只涨停股所属概念 | 综合评分排名结果 |
| 3 | 龙虎榜 | 净买额 + 买卖前3席位（含金额/亿） | ClawHub + 东财降级 |

格式：`市场代码|股票代码|数据号|值|0`

列会自动创建并设置为浅灰色，用户无需手动操作。

### 3.1 数据源总表

| 数据 | 主信源 | 接口 | 可靠性 |
|:-----|:-------|:-----|:-------|
| 涨停股票池（非ST） | 同花顺 | data.10jqka.com.cn | ✅ 高 |
| 涨停股票池（ST补充） | 东方财富 | push2ex.eastmoney.com | ✅ 高 |
| 连板天数 | TDX本地日线 | vipdoc/lday/*.day | ✅ 高 |
| 涨停原因 | 同花顺 | dataapi reason_type | ✅ 高 |
| 概念板块列表 | 东方财富 | push2(t:3) | ⚠️ 降级见下 |
| 龙虎榜（主） | ClawHub | fffy520.gicp.net | ⚠️ 中 |
| 龙虎榜（备选） | 东方财富 | data.eastmoney.com | ✅ 高 |
| 封板/炸板/跌停事件 | 东方财富 | push2ex.eastmoney.com | ✅ 高 |
| 昨日连板数据 | boards JSON缓存 | 本地文件 | ✅ 高 |

### 3.2 概念分析数据流

东方财富 push2.eastmoney.com（概念板块API）返回 `rc:102` 不可用，因此概念分析采用降级方案：

```
同花顺涨停池 → 提取 reason_type 字段
  → analyze_concepts_by_reason(up_stocks)
  → _rank_concepts_composite（V4综合评分：6维度）
  → TOP3 概念 → RDGN/HYRD/RDGN3 + extern_user.txt
```

**V4综合评分（6维度）**：

| 维度 | 权重 | 满分条件 |
|:-----|:----:|:---------|
| 广度B（涨停聚集度） | 25% | 5只涨停即满分 |
| 高度H（最高连板） | 25% | 6板即满分 |
| 梯队T（板数覆盖层数） | 15% | 3层即满分 |
| 资金M（成交总额） | 20% | 百分位排名 |
| 持续性P（历史N日TOP10） | 10% | 持续上榜加分 |
| 强度I（涨停时间中位值） | 5% | 越早封板越高 |

评分分级：

| 等级 | 评分区间 | 标签 |
|:-----|:---------|:-----|
| S级 | ≥ 70 | 主线 |
| A级 | 50-70 | 次线 |
| B级 | 30-50 | 观察 |
| C级 | < 30 | — |

### 3.3 连板天数计算（TDX本地日线）

```
今日涨停股 → 读取TDX vipdoc/lday/{market}lday/{code}.day
→ 从今日向前逐日遍历 → 涨停价公式判断 → 数连板天数
→ ST股：查昨日boards JSON继承
```

**本地涨停判断标准**：

| 市场类型 | 涨停阈值 | 说明 |
|:---------|:---------|:-----|
| 主板（60/00开头） | 涨幅 ≥ 9.8% | 对应10%涨跌幅限制 |
| 创业板（30开头） | 涨幅 ≥ 19.5% | 对应20%涨跌幅限制 |
| 科创板（68开头） | 涨幅 ≥ 19.5% | 对应20%涨跌幅限制 |
| ST/*ST | 涨幅 ≥ 4.8% | 对应5%涨跌幅限制 |

### 3.4 龙虎榜双源降级

```
主源：ClawHub API (fffy520.gicp.net:8003)
  ↓ 超时/非200时自动切换
备选：东方财富龙虎榜 API (data.eastmoney.com)
```

### 3.5 数据验证标准

**内部一致性**：
- R1: 所有连板 = 2板 + 3板 + 4板 + 5板以上之和
- R2: 涨停总数 ≥ 首板 + 连板
- R3: 最高板 ≥ 分类隐含最高板

**跨数据源交叉验证**：
- R4: 东方财富封涨停数 ≤ 同花顺涨停数
- R5: 东方财富ST涨停补充数量是否合理
- R6: 东方财富全接口数据是否完整

## 五、授权与订阅体系

### 5.1 授权方式

- **混合授权**：首次激活需联网验证，日常使用离线验证
- **每个激活码最多绑定 2 台设备**
- **激活码格式**：LICENSE-XXXX-XXXX-XXXX-XXXX

### 5.2 订阅周期

| 阶段 | 时长 | 功能 |
|:-----|:-----|:-----|
| 试用版 | 30日 | 全功能，从激活当日开始计算 |
| 正式版 | 按年（365天） | 全功能 + 微信推送 |
| 到期后 | — | 降级为试用版，每日更新暂停，保留数据查看 |

续费不影响已绑定的设备和配置。

### 5.3 用户中心

| 页面 | 功能 | 开发方 |
|:-----|:------|:-------|
| `/` | 产品介绍 + 下载链接（无需登录） | hermes |
| `/buy` | 支付宝当面付扫码付款 | hermes |
| `/activate` | 输入激活码 → 绑定设备 | hermes |
| `/manage` | 查看到期日 / 绑定设备列表 / 解绑 / 续费 | hermes |
| `/wechat-bind` | 扫码绑定微信 → 接收每日推送 | hermes |

### 5.4 授权验证流程

```
首次启动：
  ├── 有无 license.dat？→ 有：直接验证签名
  ├── 无 → 是否已输入激活码？
  │   ├── 是 → 联网验证 → 存 license.dat
  │   └── 否 → 创建30天试用记录 → 进入主界面

日常启动：
  └── 读取 license.dat → 验证签名 + 时间 + 设备数
      ├── 通过 → 正常运行
      ├── 距到期≤7天 → 弹窗提醒续费 + 继续用
      └── 已过期 → 弹窗提示续费，关闭后降级为试用版

命令行模式（定时任务）：
  └── license.verify()
      ├── 通过 → 正常执行更新
      ├── 距到期≤7天 → 正常执行更新 + 日志提示续费
      └── 已过期 → 跳过更新，日志记录"授权已过期"
```

### 5.5 支付流程

```
用户中心 /buy 页面
  → 选择「购买一年」→ 弹出支付宝收款码
  → 用户扫码付款
  → 支付宝回调通知用户中心
  → 自动生成激活码并显示在页面上（或通过微信发送）
  → 用户复制激活码 → 在连板龙中输入 → 激活成功
```

**起步方案**：支付宝当面付 + 手动发码过渡，用户超50人后上自动化发码。

### 6.2 授权模块设计（license.py）

```python
class LicenseManager:
    def activate(code, device_fingerprint)
```

### 6.4 二进制文件格式

**blocknew.cfg**（120字节/条）：
```
字节 0-47:  板块显示名 (GBK, 补零到48字节)
字节 48-49: \0\0 (固定2字节标记)
字节 50-?:  文件名 (ASCII, 不包含.blk后缀)
每条固定: 120 字节
最后追加: 48 字节全零作为终结块
```

**blocknew.clr**（320字节/条）：
```
字节 0-47:  板块显示名 (GBK, 补零)
其余所有颜色偏移位置: 全部归零 (默认白色)
```

**gridtab.dat**（38字节/条）：
```
固定38字节记录，含RID+显示名+索引
```

**.blk文件**（7字节/条）：
```
7字符ASCII：市场(1=沪/0=深) + 6位股票代码
```

## 七、安全加固（V3.1 保留）

### 7.1 SSL 网络安全

- HTTPS 默认验证证书
- HTTP URL 自动跳过SSL
- 已知问题 API 可单独关闭验证

### 7.2 文件写入安全

- 所有配置文件写入采用原子操作（.tmp → os.replace）
- gridtab.dat 写入前自动备份
- .blk 空板块写0字节（不删除文件）

### 7.3 代码加密保护

| 保护层级 | 模块 | 方式 |

| 🔒🔒 业务逻辑 | app.py / utils.py | PyArmor 混淆 |
| 🔒 辅助模块 | install.py / report.py | PyInstaller |

### 7.4 敏感信息管理

| 授权信息 | license.dat | 设备指纹绑定 |
