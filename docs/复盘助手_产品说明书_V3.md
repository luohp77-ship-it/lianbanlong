> **产品形态**：通达信AI智能复盘插件
> **交付方式**：本地安装 + 定时自动更新（双轮制）
> **收费模型**：免费使用

> **日期**：2026-06-03
> **状态**：已实现（基于 V2.2.0 + QA 审核修复）

## 一、产品定位

### 一句话定义

> 一个免费、开箱即用的通达信复盘插件，每天收盘后自动更新自定义板块，用户打开通达信即可看到涨停梯队、龙虎榜、热点概念全景。

### 解决的用户痛点

| 痛点 | 本产品方案 |
|:-----|:----------|
| 收盘后需手动翻涨停列表、分类连板 | 自动获取全部涨停数据，按连板天数分类写入板块 |
| 不知道今天哪个概念最强 | 综合评分排名（涨停数+强度+资金），动态命名热点概念 |
| 龙虎榜需切换网站查看 | 自动拉取写入同一板块 |
| 跌停/炸板/曾涨停容易忽略 | 自动监控并分类写入对应板块 |
| ST股涨停被主流数据源遗漏 | 双数据源互补：同花顺(非ST) + 东方财富(ST) |
| 看不到涨停原因和所属概念 | 自定义数据列：extern_user.txt 写入涨停原因+所属概念+龙虎榜 |
| 没有系统化复盘报告 | 每天自动生成Word报告，含封面+情绪+涨停+龙虎榜+梯队诊断 |
| 复盘工作重复枯燥 | 全自动，打开通达信直接看 |

### 目标用户

- A 股散户，有每日复盘习惯
- 使用通达信（含券商定制版）
- 关注涨停板、连板效应、龙虎榜、概念热点
- 使用龙头战法复盘

### 平台策略

**本产品仅支持通达信**。不做东方财富、大智慧等其他交易软件适配。

### 2.2 标签栏动态命名规则

```
gridtab.dat 名称字段：14字节(GBK)

连板板块：保持固定名（昨涨停/首板/2连板/...）
概念板块：{数字}{概念名}，如"1消费电子""2电力改革""3算力通信"
  - 数字1=主线、2=次线、3=潜在
  - 名称不带括号、不带涨停数
  - 长度控制：自动截断确保 ≤14 字节
```

### 2.3 自定义数据列（extern_user.txt）

| 数据号 | 名称 | 内容 | 数据源 |
|:-------|:-----|:-----|:-------|
| 1 | 涨停原因 | 每只涨停股的涨停原因 | 同花顺 reason_type |
| 2 | 所属概念 | 每只涨停股所属概念 | 综合评分排名结果 |
| 3 | 龙虎榜 | 净买额 + 买卖前3席位（含金额/亿） | ClawHub + 东财降级 |

格式：`市场代码|股票代码|数据号|值|0`

**列会自动创建并设置为荧光绿色**（`set_column_green`），用户无需手动操作。首次运行后通达信中即可看到三列自定义数据。

### 2.4 Word复盘报告（每日自动生成）

报告保存路径：`reports/复盘报告_YYYY-MM-DD.docx`

| 章节 | 内容 |
|:-----|:-----|
| 封面 | 日期、涨停/跌停/最高板/龙虎榜/生命周期总览 |
| 一、情绪指数 | 溢价率、炸板率、晋级率 + 生命周期 + 文字解读 |
| 二、涨停数据 | 全部涨停股，按3板→2板→首板分组表格 |
| 三、龙虎榜 | 全部龙虎榜个股，含净买额 + 买卖前5席位 |

| 六、最高板 | 最高板龙头详情（原因/概念/龙虎榜） |
| 七、市场梯队诊断 | 每板级为锚点：含代码/名称/原因/概念/龙虎榜 |

### 3.2 概念分析数据流

东方财富 push2.eastmoney.com（概念板块API）返回 `rc:102` 不可用，因此概念分析采用降级方案：

```
同花顺涨停池 → 提取 reason_type 字段
  → analyze_concepts_by_reason(up_stocks)
  → _rank_concepts_composite（综合评分）
  → TOP3 概念 → RDGN/HYRD/RDGN3 + extern_user.txt
```

**综合评分公式**：
```
综合评分 = 涨停分 × 40% + 强度分 × 35% + 资金分 × 25%

涨停分 = 概念内涨停股数 / 最高概念涨停数 × 100
强度分 = min(最高板×2 + 梯队数×3, 30) / 30 × 100
资金分 = 板块成交额 / 最高成交额 × 100
```

### 3.3 连板天数计算（TDX本地日线）

```
今日涨停股 → 读取TDX vipdoc/lday/{market}lday/{code}.day
→ 从今日向前逐日遍历 → 涨停价公式判断 → 数连板天数
→ ST股：查昨日boards JSON继承
```

**本地涨停判断标准**（使用涨停价精确公式，非百分比近似）：

| 市场类型 | 涨停阈值 | 说明 |
|:---------|:---------|:-----|
| 主板（60/00开头） | 涨幅 ≥ 9.8% | 对应10%涨跌幅限制 |
| 创业板（30开头） | 涨幅 ≥ 19.5% | 对应20%涨跌幅限制 |
| 科创板（68开头） | 涨幅 ≥ 19.5% | 对应20%涨跌幅限制 |
| ST/*ST | 涨幅 ≥ 4.8% | 对应5%涨跌幅限制 |

### 3.4 龙虎榜双源降级

```
主源：ClawHub API (fffy520.gicp.net:8003)
  ↓ 超时/非200时自动切换
备选：东方财富龙虎榜 API (data.eastmoney.com)
```

### 3.5 数据验证标准

**内部一致性**：
- R1: 所有连板 = 2板 + 3板 + 4板 + 5板以上之和
- R2: 涨停总数 ≥ 首板 + 连板
- R3: 最高板 ≥ 分类隐含最高板

**跨数据源交叉验证**：
- R4: 东方财富封涨停数 ≤ 同花顺涨停数
- R5: 东方财富ST涨停补充数量是否合理
- R6: 东方财富全接口数据是否完整

## 四、每日更新流程

### 4.1 时间线（双轮更新制）

```
第一轮 15:26 Windows计划任务触发（收盘更新）
├── 1. 获取涨停数据（同花顺，含reason_type）
├── 1b. 获取东方财富数据（ST补充+封板/跌停/炸板）
├── 合并涨停池 = 同花顺 + 东财ST
├── 2. 连板天数计算（TDX本地日线，带LRU缓存加速）
│   ├── 非ST：读TDX日线 → 涨幅判断 → 数连板
│   └── ST：boards JSON继承
├── 3. 东方财富封板/跌停/炸板
├── 4. 概念分析（reason_type + 综合评分 + 允许重叠）
├── 5. 数据校验（内部一致性 + 跨源交叉验证）
├── 6. 安装板块配置（含动态热点命名）
├── 7. 写入.blk文件 + extern_user.txt（含数据号1/2/3）
├── 8. 更新DBKPJM.sp看盘界面概念名称（含异常保护）
└── 用户15:25后重启通达信 → 涨停/概念数据已就绪

第二轮 18:00 Windows计划任务触发（全量更新）
├── 重复第一轮全部步骤
├── + 龙虎榜数据（ClawHub主源 + 东财备选降级，此时已公布）
├── + 生成Word复盘报告 → reports/复盘报告_YYYY-MM-DD.docx
└── 用户18:00后重启通达信 → 数据完整（涨停+龙虎榜+Word报告）
```

### 4.2 更新方式

| 场景 | 触发方式 | 说明 |
|:-----|:---------|:-----|
| 15:26 自动 | Windows计划任务 | 涨停+概念+板块，龙虎榜未完成（后台静默运行） |
| 18:00 自动 | Windows计划任务 | 全量：涨停+龙虎榜+Word报告（后台静默运行） |
| 手动更新 | 运行 `python app.py --update` | 任意时刻手动刷新（数据就绪时使用） |

**用户侧零操作**：无需点击任何 .bat 文件，保持通达信原始快捷方式不变。15:25后重启通达信即自动获取最新数据。

### 4.3 安装流程

```
用户双击 install.exe
├── 1. 查找通达信路径
│   ├── 注册表查找
│   ├── C:\D:\E:\new_tdx 默认路径
│   └── 手动输入
├── 2. 保存配置
├── 3. 安装板块（写入cfg/clr/dat/blk）
├── 3b. 安装看盘界面（DBKPJM.sp → T0002/pad/）
├── 3c. 设置自定义数据列颜色（荧光绿，含数据号1/2/3）
├── 4. 创建定时任务（15:26 + 18:00 双轮）
├── 5. 首次运行数据更新
└── 提示：
    "安装完成！重启通达信即可使用。
     自定义数据列（涨停原因/所属概念/龙虎榜）已自动创建。
     如需导入看盘界面：功能→定制版面→导入版面"
```

### 5.3 二进制文件格式

**blocknew.cfg**（120字节/条）：
```
字节 0-47:  板块显示名 (GBK, 补零到48字节)
字节 48-49: \0\0 (固定2字节标记)
字节 50-?:  文件名 (ASCII, 不包含.blk后缀)
每条固定: 120 字节
最后追加: 48 字节全零作为终结块
```

**blocknew.clr**（320字节/条）：
```
字节 0-47:  板块显示名 (GBK, 补零)
其余所有颜色偏移位置: 全部归零 (默认白色)
```

**gridtab.dat**（38字节/条）：
```
固定38字节记录，含RID+显示名+索引
```

**.blk文件**（7字节/条）：
```
7字符ASCII：市场(1=沪/0=深) + 6位股票代码
```
