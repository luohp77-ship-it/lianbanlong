## 一、安装与首次体验（用户对产品的第一印象）

### 问题1：安装后通达信板块为空，用户不知道需要手动刷新

**现状**：install.py 安装完成后提示"重启通达信查看板块"，但通达信重启后**并非自动显示新板块数据**。用户需要在通达信内**手动点击板块列表的"刷新"按钮**才能看到自定义板块。文档中**完全没有提及这一步**。

**用户感受**：装完后打开通达信看到的是一片空白标签——第一反应是"这东西是不是坏的？"

**建议**：
- 安装完成提示改为："重启通达信后，请在板块列表中**右键→刷新**查看自定义板块"
- 条件允许的话，研究是否可以通过模拟按键/发送消息让通达信自动刷新
- 至少在"常见问题"中把这条放在第一条

### 问题2：16:30 + 18:30 双时间点方案存在逻辑漏洞

**现状**：产品说明书 V3 第 371 行要求"18:30 龙虎榜补充"，但代码中只注册了**一个** 16:30 的定时任务。引擎运行后 `latest.json` 标记了当天日期，launcher 检查发现"今天已更新"就跳过。

**问题**：18:30 谁来触发龙虎榜更新？目前这段逻辑**完全不存在**。

**建议**：
- 方案A：注册两个定时任务（16:30 主更新、18:30 仅更新龙虎榜）
- 方案B：在 launcher 中增加"数据是否完整"判断（检查 latest.json 中的 lhb 字段），不完整时仍触发更新
- 最简单方案：取消分两次更新的设计，统一在 18:30 更新所有数据（龙虎榜确实18:00后才完整）

### 问题3：数据源分歧问题——同花顺和东方财富涨停数不一致

**日志证据**：从 `logs/20260601.log` 可见：
```
涨停(同花顺): 49只
涨停(东财): 18只
```

**49 vs 18 的差距非常显著**。两个数据源对"什么是涨停"的定义不同，但用户不知道以谁为准。代码实际使用的是同花顺的数据，但概念分析用的是东方财富的行业分类——**如果一只股票在同花顺涨停列表中但不在东方财富的封涨停列表中，它会被写入 .blk 但不会参与概念排名**。

**建议**：
- 明确数据优先级：同花顺涨停池为主，东方财富封板数据为辅
- 在概念分析中，应该用同花顺的涨停列表（而非东方财富的封涨停列表）去匹配东方财富的行业成分股
- 在日志中标注两者差异，超过30%差异时告警

## 二、核心功能缺失（用户复盘真正需要但产品没做的）

### 问题4：概念分析用的是"行业板块"而非"概念板块"

**这是产品设计与实际实现的根本性错位。**

知识库 `概念与短线连板知识体系.md` 第一条就强调：
> "概念 ≠ 行业板块。短线炒的是概念不是行业。同花顺中看'概念板块'比看'行业板块'更重要。"

但 `concept_analyzer.py:20` 的 API 请求是：
```python
fs=m:90+t:2   # t:2 = 行业板块
```

应该是 `t:3`（概念板块）才对。东方财富的概念板块包含"电力改革""免税概念""算力通信"等真正的短线炒作概念，而行业板块是"电力""商贸零售"等固定分类。

**用户感受**：我是来看"电力改革"概念今天有多热的，结果你告诉我"电力行业"涨停8只——这是两个完全不同的东西。

**建议**：
- 立即将 `t:2` 改为 `t:3` 获取真正的概念板块
- 保留 `t:2` 行业数据作为备选/补充维度
- 概念映射表以 t:3 的概念名为 key 重新建立

### 问题5：缺少短线复盘的三个核心指标

对于一个用"龙头战法"做复盘的散户，每天收盘后最关心的三个指标是：

| 核心指标 | 含义 | 是否实现 |
|:---------|:-----|:--------:|
| **晋级率（一进二成功率）** | 昨日首板今天晋级2板的比例 | ❌ 未实现 |
| **溢价率** | 昨日涨停股今日开盘平均涨幅 | ❌ 未实现 |
| **炸板率** | 今日冲板后开板的股票占比 | ❌ 未实现 |

知识库 `龙头战法完整体系与每日复盘模板.md` 反复强调这些是判断情绪周期的核心指标：
- "首板封板率是否 > 70%？"——判断发酵期
- "晋级率下降，封板率下降"——分歧期信号
- "低位首板、1进2的封板率高、溢价高、晋级率高"——发酵期特征

**但产品完全没有计算或展示这些指标。**

**建议**：
- P0：增加"昨日涨停股今日表现"追踪（引擎运行时自动对比昨日和今日数据）
- P1：增加晋级率计算（昨日首板/2板/3板各层级今天的晋级比例）
- P1：增加炸板率（今日炸板/今日曾涨停）

### 问题6：龙虎榜只有前20净买入排名，缺少游资分析

对于使用龙头战法的散户，龙虎榜最大的价值是看**游资动向**：今天哪些游资在买、哪些在卖、席位是否有联动。

目前产品只做了净买入前20排序，这等于只展示了最表层的龙虎榜信息。

**建议**（远期规划）：
- 标记知名游资席位（如炒股养家、方新侠、作手新一等的常用席位）
- 统计游资合力（同一只股票多家游资同时买入 → 强信号）
- 统计机构vs游资博弈（机构净买+游资净卖 or 反之）

## 三、功能可用性改善

### 问题7：概念板块"互斥展示"导致次线和潜在板块寒酸

设计意图是好的：三个概念板块"互斥展示"，同一只股票不出现在多个板块。但实际效果是：

- **主线** = 大消费（9只涨停股）——看着很丰富
- **次线** = 电力改革（8只涨停股 - 与大消费重叠的3只）——只剩5只，看着像次要
- **潜在** = 房地产（5只涨停股 - 与前两个重叠的2只）——只剩3只，几乎看不见

而实际上"电力改革"的热度本就是8只涨停，被"互斥"削弱了视觉说服力。

**建议**：
- 在 .blk 板块展示中允许重叠（每只股票可以属于多个概念板块）
- 将"互斥"规则仅用于**排名阶段**避免概念重复计算
- 或在板块名称中同时显示全量和互斥计数："次线:电力改革(8只/5只独有)"

### 问题8：概念映射表不够灵活，新概念无法及时更新

`standalone/concept_map.json` 只有 30 条映射。实际市场概念远不止这些。例如"商业航天""低空经济""合成生物"等新概念出现时，映射表中没有，就会以原始行业名显示——而原始行业名可能对用户不友好。

**建议**：
- 映射表增加"模糊匹配"规则（如行业名包含"零售"→自动归"大消费"）
- 或在程序启动时静默拉取最新版映射表（从 GitHub Pages 下载）
- 至少将映射表扩充到 80-100 条，覆盖当前主流的轮动概念

### 问题9：客户端界面体验

**现状**：`app.py` 使用 tkinter 构建桌面APP，UI 风格非常基础。虽然功能完整，但视觉上像是 2005 年的软件。

**建议**：
- 短期：优化 tkinter 的主题配色（可以参考 report.py 的 HTML 暗色主题设计）
- 中期：考虑用 Electron + React 重写桌面端，或至少用 PyQt/PySide 替换 tkinter
- 或者：放弃独立桌面APP，全部功能集成到 HTML 报告中，用户用浏览器查看

## 四、可靠性与健壮性

### 问题10：ClawHub 龙虎榜服务可靠性存疑

数据源 `http://fffy520.gicp.net:8003/api/lhb/daily` 使用动态 DNS，意味着：
- 如果服务部署在家庭宽带，IP 可能频繁变更
- 动态 DNS 服务本身可能不稳定
- 没有 HTTPS，数据在传输中可以被篡改

**建议**：
- P0：接入东方财富龙虎榜 API 作为备选方案
- P1：考虑将龙虎榜数据迁移到更稳定的托管方案（云服务器/Serverless）

### 问题11：TG Bot token 硬编码在源代码中

`tg_bot.py:12`:
```python
TOKEN = '8747950702:AAHDSMjzhkjzhVF6d-rpj7d5Y6TqdmLfLBY'
```

这是一个安全漏洞：任何人拿到源代码就能控制这个 Bot。

**建议**：
- 立即将 token 移至环境变量或 config.json
- 如果代码已经提交到公开仓库，应立即重置 Telegram Bot Token

## 五、文档与产品的对齐

### 问题12：文档版本体系混乱，对用户认知造成困扰

| 文档 | 自称版本 | 实际代码完成度 | 冲突 |
|:-----|:--------|:-------------|:-----|
| 产品策划书 V1 | V1.0 | 已废弃 | — |
| 产品策划书 V2 | V2.0 | 部分实现 | 收费方案已废弃 |
| 产品说明书 V3 | V3.0 | 大部分实现 | 标注"已采纳20/22"但审核意见验证为16条未实现（现已大部分修复） |
| config.json | 无版本字段 | — | — |
| app.py | 显示"v1.0" | V3级功能 | 代码比文档新 |

**同一个产品，文档有三个版本号**。新用户看到这些会完全困惑。

**建议**：
- 保留 `产品说明书_V3.md` 作为唯一官方文档，删除或归档旧的 V1/V2 文档
- 在 config.json 和 app.py 中统一版本号
- V3 文档中"已采纳 20/22"的标注现在基本属实了，但建议改为"V3 已实现的功能清单"

### 问题13：用户文档缺失——没有"使用教程"

目前 docs 目录只有产品策划书、产品说明书、审核意见——全是开发/策划视角的文档。

**一个真实的用户下载安装包后，没有任何指引文档告诉他：**
- 安装后通达信里怎么找到这些板块？
- 每个板块怎么看？（首板表示今天第一次涨停、2连板表示连续两天涨停……）
- "主线/次线/潜在"是什么意思？与龙头战法的关系？
- 电脑关机了定时任务还会跑吗？（不会，因为 Windows 计划任务需要电脑开机）

**建议**：
- 创建一个 `用户使用手册.md`，纯面向最终用户
- 内容包括：安装步骤（带截图）、板块含义、常见问题、数据更新时间说明
- 不需要专业知识，不要用"API""数据源"这类开发术语
